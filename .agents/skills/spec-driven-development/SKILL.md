# Spec-Driven Development Skill

Use this skill whenever tasked with implementing non-trivial features, refactoring core abstractions, or creating new system components.

## Workflow

1. **Requirements & Scope Definition**:
   - Explicitly list the functional requirements, non-functional constraints (memory, latency, hardware limits), and interface specifications.
   - Identify existing codebase patterns and dependencies to reuse instead of reinventing.

2. **Architecture & Design Specification**:
   - Define data structures, API signatures, state transitions, and component boundaries.
   - Detail edge-case handling (missing files, invalid inputs, hardware fallbacks).

3. **Task Decomposition & Verification Plan**:
   - Break implementation down into small, verifiable sub-tasks.
   - Define concrete verification steps for each task (unit test, assertion, integration script).

4. **Execution & Validation**:
   - Execute tasks sequentially.
   - Verify each task against the predefined criteria before declaring complete.
